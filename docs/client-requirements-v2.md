## 1. نظام الإنذارات والتحصيل (Payment Warning System)

### الوصف
نظام تنبيهات تلقائي للمستحقات المتأخرة على ثلاث مراحل:

| المرحلة | التوقيت | النوع |
|---------|---------|------|
| إنذار أول | بعد 15 يوم من رسالة الاستحقاق الأولى | تلقائي |
| إنذار ثاني | بعد شهر من الاستحقاق الرئيسي (أو 15 يوم من الإنذار الأول) | تلقائي |
| خطاب مالي / إخلاء | المرة الثالثة | يدوي |

### المتطلبات
- عند إصدار الخطاب الثالث (يدوي): إمكانية إضافة المستأجر إلى البلاك ليست
- عرض حالة الإنذارات في تفاصيل العقد
- فلترة العقود حسب حالة الإنذار

### التغييرات المطلوبة
- **Model جديد:** `PaymentWarning` (leaseId, scheduleItemIndex, warningLevel, sentDate, type)
- **Lease Model:** إضافة `warnings: PaymentWarning[]`
- **UI:** عرض الإنذارات في تفاصيل العقد + أيقونات تنبيه في قائمة العقود

### الوصف
تحسين نموذج المباني وواجهة المستخدم.

### الحقول الموجودة فعلاً في الـ Model ✅
- `imageUrl` - صورة المبنى
- `deedImageUrl` - صورة صك المبنى

- تعديل `ownerManagerName` من قبل المدير فقط (صلاحية)
- واجهة إضافة/تعديل العدادات مع اختيار النوع

### التغييرات المطلوبة

- **Building Detail:** إضافة عرض الخريطة وتفاصيل العدادات

## 8. تحديثات عقد الإيجار (Lease Contract Updates) - الأهم

### الوصف
تحديثات جوهرية على نظام إنشاء وإدارة العقود.

### 8.1 حقول جديدة في العقد
| الحقل | النوع | ملاحظة |
|-------|------|--------|
| `buildingId` | string | اختيار المبنى أولاً |
| `createdByUserId` | string | اليوزر الذي أنشأ العقد |
| `createdByUserName` | string | اسم منشئ العقد |
| `assignedToUserId` | string | اليوزر المسؤول عن متابعة العقد |
| `assignedToUserName` | string | اسم المسؤول |
| `contractDuration` | enum | سنة، 6 أشهر، 3 أشهر |
| `totalContractValue` | number | **يحل محل `rentAmount`** - قيمة العقد الكلي |
| `rentalCommission` | number | عمولة التأجير (2.5% افتراضي) |
| `commissionPercentage` | number | نسبة العمولة (قابلة للتعديل) |
| `commissionDiscount` | number | خصم على العمولة (حتى 100%) |
| `depositPaid` | boolean | هل تم دفع التأمين |
| `commissionPaid` | boolean | هل تم دفع العمولة |
| `ownerManagerName` | string | مسؤول المبنى مع المالك |
| `renterManagerName` | string | مسؤول المبنى مع المستأجر |

### 8.2 استبدال قيمة الإيجار بقيمة العقد الكلي
- **قبل:** `rentAmount` (قيمة الإيجار الشهري/الدوري)
- **بعد:** `totalContractValue` (قيمة العقد الكلية) → يتم تقسيمها حسب دورة الدفع

### 8.3 إزالة تاريخ الاستحقاق
- إزالة `dueDayOfMonth`
- تاريخ الاستحقاق يعتمد على `startDate` تلقائياً

### 8.4 منطق دورة الدفع حسب مدة العقد

| مدة العقد | خيارات دورة الدفع |
|-----------|-------------------|
| سنة | شهري، ربع سنوي، كل 6 أشهر، سنوي |
| 6 أشهر | شهري، كل 3 أشهر، دفعة كاملة |
| 3 أشهر | شهري، دفعة كاملة |

### 8.5 عمولة التأجير
- تحسب تلقائياً: 2.5% من قيمة العقد (غير شاملة الضريبة)
- قابلة للزيادة فوق 2.5%
- قابلة للخصم (حتى 100%) ضمن صلاحية المدير فقط
- العقد يبقى **غير نشط** حتى يتم دفع التأمين والعمولة

### 8.6 واجهة إنشاء العقد (Lease Wizard)
1. اختيار المبنى → عرض الوحدات المتاحة فقط
2. بحث واختيار المستأجر
3. تحديد مدة العقد (سنة / 6 أشهر / 3 أشهر)
4. تحديد تاريخ البداية → النهاية تحسب تلقائياً
5. إدخال قيمة العقد الكلية
6. اختيار دورة الدفع (حسب مدة العقد)
7. حساب العمولة تلقائياً مع إمكانية التعديل
8. إدخال مبلغ التأمين
9. توليد جدول الاستحقاقات تلقائياً

### 8.7 فلاتر إضافية
- فلتر عقود لم يتم دفع تأمينها
- فلتر عقود لم يتم دفع عمولة التأجير

### 8.8 تفاصيل العقد
- عرض مسؤول المبنى مع المالك
- عرض مسؤول المبنى مع المستأجر
- عرض اسم منشئ العقد
- عرض حالة العقد مع توضيح سبب عدم النشاط
